What Are Signals?

Signals are a modern, efficient solution for reactive state and UI management in JavaScript frameworks, offering automatic dependency tracking and targeted updates.

What Are Signals?

Let's try to define them

Signals are Pub Sub
Signals are an event system
Signals are a reactive primitive
Signals have a tc39 proposal (stage 0): github.com/tc39/proposal-signals
Signals are not new, just reimagined ideas going back to knockout in 2010

batch(fn)

The batch(fn) function can be used to combine multiple value updates into one "commit" at the end of the provided callback. Batches can be nested and changes are only flushed once the outermost batch callback completes. Accessing a signal that has been modified within a batch will reflect its updated value. 


							const name = signal('Jane');
							const surname = signal('Doe');

							// Combine both writes into one update
							batch(() => {
								name.value = 'John';
								surname.value = 'Smith';
							});
